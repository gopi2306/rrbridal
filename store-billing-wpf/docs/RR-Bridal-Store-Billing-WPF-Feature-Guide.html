RR Bridal Store Billing (WPF) — End-to-End Feature Guide

Document version: 2.0  |  Last updated: 16 July 2026
Application: RRBridal.StoreBilling.App (TruBilling POS)
Purpose: Offline-first retail POS for RR Bridal stores with central sync
Audience: Support engineers, store IT, and managers who need a technical feature reference

1. Application Overview

The WPF Store Billing application is the point-of-sale client for RR Bridal retail counters. Bills, returns, quotations, credit collections, and day-close data are saved to local MongoDB and pushed to the central NestJS API when connectivity is available.

  • Each till uses unique DEVICE_ID and POS_COUNTER in the .env file beside the executable.
  • POS Counter 1 is the primary (manager) till with full navigation and auto-sync.
  • Secondary counters can bill, quote, return, look up bills, close day, and print barcodes, but cannot open Settings, Dashboard, Ledger, Analytics, Online Sales, Credit Bills, or Expenses.
  • Product master (including mediaItems), stock, promotion schemes, salesmen, barcode label designs, and users are pulled from central; sales events are pushed via an outbox queue.

2. Login, Session and Security

2.1 Cashier login (local)

  • Email + password against store_users in local MongoDB (synced from central).
  • BCrypt verification; one active session per user per DEVICE_ID (machine lock).
  • Logout via header icon or navigation menu; returns to login screen.

2.2 Central API login (Settings)

  • Used for sync, receipt profile pull context, WhatsApp API, and company profile.
  • Settings → Connection & sync: email/password login; token stored in central_auth.json.

2.3 Per-user limits

  • maxDiscountPercent caps combined item % and cash discount at bill post.
  • Till number (POS_COUNTER) controls navigation access, not the user role field (role is synced but not enforced in UI).

3. Main Shell and Navigation

Header: logo, store name, page title, global search, day status chip, user badge, notifications bell, settings gear (primary), logout. Left hamburger opens full navigation. Footer shows keyboard shortcuts. F2 New is page-aware (clears/resets the current page).

4. Billing (POS)

4.1 Customer section

  • Fields: customer code, phone (10-digit required to post), name, salesman (selectable dropdown).
  • Salesman is required at post when at least one active salesman exists in the store master.
  • Find Customer dialog; phone auto-search; quick capture for walk-ins.
  • Navigate to Customers / Salesman pages to register and return to the bill.

4.2 Bill flags

  • Hold bills, Door delivery, Online COD, Stitching (+ delivery date), Print invoice.

4.3 Line items

  • Entry row: SKU/barcode + Enter adds line; name opens product pick list.
  • Lookup from local product cache; Add manual line; Import CSV is a stub (not implemented).
  • Configurable columns: Minimal / Standard / Full (Settings → Other).
  • Product pick list always shows SKU, Name, Rate, and Image note (first non-empty mediaItems[].description).
  • Stock validation at post; stock decremented on successful post.
  • Optional confirm when adding a product already on the bill (Settings → Other).

4.4 Discounts and totals

  • Item discount %, cash discount, promotion schemes, round off, inter-state IGST toggle.
  • Apply customer credit notes; Online COD cannot combine with credit note.
  • Credit billing (pay-later): advance at post when enabled in Settings; balance collected later on Credit Bills.

4.5 Hold, post, and payment

  • F8 Hold — save draft with hold number (no stock change).
  • F9 Post — requires name, phone, lines, open business day; assigns bill number.
  • Payment dialog: Cash, Card (Pine Labs), UPI (Razorpay POS), Credit note, Split, Credit advance, Online COD (no payment at post).
  • Optional WhatsApp auto-send and print after post.
  • F10 — print stub without posting.

5. Quotations

  • Quotations list: search, open, cancel; create new quotation.
  • Editor reuses billing-style entry (customer, lines, discounts) and saves a local quotation document (not a posted bill).
  • Only open quotations can be converted to Billing (LoadFromQuotationDocument).
  • Available on all tills.

6. Customers and Salesmen

6.1 Customers

  • Auto-generated customer code; saves locally and syncs to central API.
  • Required: name, mobile. Optional: address, GSTIN, email, credit customer flag.
  • F4 Save, Escape Cancel on Customers page.

6.2 Salesmen

  • Dedicated Salesman page: register and manage store salesmen used on billing.
  • Salesmen are also pulled from central during sync.

7. Credit Bills (primary till)

  • Search pay-later / credit sales by status (pending, partial, settled), customer, date.
  • Record balance collection: Cash, Card, UPI, Credit note, Split (subject to Settings credit rules).
  • Print credit balance receipt (format configured separately: Thermal or A4).
  • Dashboard Overview links to pending credit bills.

8. Bill Lookup

  • Unified search for posted bills (invoice no., customer, mobile, dates).
  • From a selected bill: view detail, start return, start adjustment, duplicate print, credit-note print.
  • Available on all tills; complements standalone Returns / Adjustments / Duplicate pages.

9. Sale Returns

  • Lookup bill by invoice number or last 3–4 digits.
  • Return mode: credit note or cash refund.
  • Exchange lines with payment for difference; stock updated; thermal receipt for exchange.
  • Settings → Other: Allow multiple returns per bill enables partial returns across transactions with cumulative returned qty tracking.

10. Adjustments

  • Load posted bill; edit adjusted qty/rate; reason required; day must be open.

11. Duplicate Print

  • Bill tab: search and reprint with DUPLICATE watermark; Send WhatsApp.
  • Credit Note tab: search and reprint linked sale-return receipt.
  • F11 shortcut; can be disabled in Settings → Other.

12. Day Close and Cash Management

  • Open day with opening cash; live summary of bills, tender, expenses, movements.
  • Cash deposit and withdrawal entries (today, while day open).
  • Cash hand-over with denomination counting; close day; F3 print hand-over slip.
  • Download full report (CSV or Excel).
  • Daily expenses (primary till only).

14. Barcode Label Printing

  • Add products by SKU/barcode; F6 pick list; set print qty per line.
  • Label design synced from central; F5 print labels (TSPL/ZPL); F7 clear.
  • Price columns follow store/selling price rules from local cache.

15. Settings (primary till)

15.1 Connection and sync

  • Central login/logout.
  • Run sync — full store sync: push outbox, pull products/transfers/schemes/users/masters, receipt config, barcode designs, salesmen.
  • Re-sync products — resets product sync cursor and paginated full catalog pull (use when prices/catalog look stale after central updates).
  • Refresh status — outbox count, cursors, last error, diagnostics, auto-sync status.

15.2 Receipt and printing

  • Store header fields; thermal and office printers; Thermal / A4 / A5 / A4 Commercial.
  • Separate credit receipt format: Thermal or A4.
  • A4 and A5 pre-printed toggles + mm alignment editors; also print thermal first.
  • Save, preview invoice, preview credit receipt.

15.3 WhatsApp

  • Auto-send bill on post; connection status and template from central; refresh status; test send.

15.4 Other

  • Allow duplicate print; confirm duplicate product add; CN remaining cash payout; multiple returns per bill.
  • Credit billing: enable; require credit customer; min advance % / amount; allow zero advance; partial collection; max balance per bill.
  • Alteration GST included vs flat charge; line item column detail level.
  • Razorpay POS: enable, username, app key, API base, device ID, poll interval, timeout.
  • Developer tests: Pine Labs ₹1, Razorpay POS, purchase intent.

16. Sync with Central Backend

  • Push: pending outbox_events (bills, returns, expenses, day sessions, COD payments, credit collections, stock transfer received, etc.) → /api/sync/push.
  • Pull: ProductUpserted, stock transfers awaiting/completed, promotion scheme upsert/delete, StoreInventoryAdjusted, plus masters, store users, salesmen, receipt profile, barcode label designs.
  • Product cursor: compound {updatedAtIso}|{objectId} so price and media updates re-pull; legacy bare ObjectId triggers one-time full catch-up.
  • Product upsert: preserves local stockQty; merges duplicate SKU stubs; if central storePrice is missing or ≤ 0, copies sellingPrice into local storePrice.
  • mediaItems: stored on product document and synced with the product payload; pick list shows description text only.
  • Periodic auto-sync on primary till when SYNC_INTERVAL_MINUTES > 0.

17. Product Media

  • Central product field mediaItems: [{ url, description? }]; legacy mediaUrls: string[] resolved on read.
  • Upload: POST /api/media/products/:productId/image multipart file + optional description.
  • Store UI: product pick list Image note column = first non-empty description.
  • ProductImageCache can download GET /api/media/products/:productId/image to disk; no thumbnail UI in billing yet.

20. Typical Daily Workflow

  1. Login with cashier credentials.
  2. Primary till: Settings → Login to central → Run sync (use Re-sync products after major central catalog/price updates).
  3. Day Close → Open day with opening cash.
  4. Billing — customer phone — salesman — add items — discounts — F9 Post — Payment (or quotation → convert → bill).
  5. Print / WhatsApp as configured.
  6. Process returns, adjustments, or credit collections as needed.
  7. Record expenses and cash movements (primary till).
  8. Day Close → Cash hand-over → Close day.
  9. Download day-close report; ensure outbox is synced.
  10. Logout.

21. Local MongoDB Collections

Core: store_bills, held_bills, quotations collections used by Quotation services, store_sale_returns, customer_credit_notes, store_credit_note_cashouts, store_adjustments, store_daily_expenses, store_cash_movements, store_day_sessions, store_customers, store_users, local_products_cache, local_stock_transfers, local_promotion_schemes, outbox_events, sync_state, local_payments, store_audit_logs, salesman collections, master collections (master_hsn_codes, master_brands, etc.).

22. Known Limitations

  • Import CSV on Billing is not yet implemented.
  • Exchange credit-note duplicate reprint has limited support.
  • User role field is synced but not enforced in UI (till number gates pages).
  • Product image thumbnails are not shown in billing; only Image note text from mediaItems.
  • ProductImageCache is registered but not wired into Views yet.
